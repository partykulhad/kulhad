Add a new "Business Unit Performance Comparison" chart to the Kulhad dashboard, showing all vending machines side-by-side in one grouped bar chart (one pair of bars per machine: Cups Dispensed + Revenue), with a date-range filter. This is a Next.js 15 + Convex + Recharts app.

## Context (already verified in the repo)

- Dashboard page: `app/(private)/dashboard/page.tsx`. The "all machines" overview branch already renders `<OverviewCards machines={machines} />` then `<SalesChart />` (full width) then status cards, `<InventoryLevels>`, and `<MachinesTable>`.
- `machines` comes from `useQuery(api.machines.list)` (`convex/machines.ts`), which returns all rows of the `machines` table (`convex/schema.ts` line ~30). Each machine has `id` (string), `name` (string — this is the display label, e.g. "Kulhad party", "Walmart", "Ikea" — these are effectively "business units"/locations), `address`, etc.
- `transactions` table (`convex/schema.ts` line ~297) has `amount`, `cups`, `amountPerCup`, `machineId` (string, matches `machine.id`, NOT the Convex `_id`), `status` ("paid" for completed sales), `createdAt` (number, epoch ms).
- `convex/transactions.ts` already exports `list` (all transactions) and `getByMachineId` (filtered by machine, indexed via `by_machineId`).
- Existing chart `components/charts/sales-chart.tsx` (`SalesChart`) already aggregates ALL machines' transactions over time (day/hour buckets) using `useQuery(api.transactions.list)`, filters `status === "paid"`, and sums `amount`/`cups`. It uses Recharts `BarChart`/`AreaChart`/`LineChart` with **two y-axes** (`yAxisId="left"` for revenue, `yAxisId="right"` for cups) — this is the exact dual-axis pattern to reuse, just grouped by machine instead of by date.
- There is currently NO per-machine comparison chart anywhere in the codebase (confirmed via grep — no "Business Unit Performance Comparison", "Cross-unit", etc.). This is a net-new component.

## What to build

Reference image: a grouped bar chart titled "Business Unit Performance Comparison" / subtitle "Cross-unit revenue and consumption analysis", with a date-range picker (e.g. "Jun 26 - Dec 19") in the header. X-axis = machine name (e.g. "APPLE Bangalore", "Ikea", "Kulhad party", "New Urban ketl", "WALMART"). Left Y-axis = Cups Dispensed (green bars), right Y-axis = Revenue ₹ (orange bars). Legend below: "Cups Dispensed" / "Revenue (₹)". Currently machines with no transactions in range still show on the axis with empty/zero bars (per the reference image, most machines are flat at 0 except one).

1. **New component**: `components/charts/machine-comparison-chart.tsx` (named export `MachineComparisonChart`), modeled after `components/charts/sales-chart.tsx` for styling/structure (same Card wrapper, gradient header, dual-axis Recharts `BarChart`, `ResponsiveContainer`, custom tooltip).
2. **Data**: pull `machines` via `useQuery(api.machines.list)` and `transactions` via `useQuery(api.transactions.list)` (both already exist, no new Convex queries needed). For each machine, filter transactions where `t.machineId === machine.id && t.status === "paid"` and optionally `t.createdAt` within the selected date range, then sum `cups` and `amount`. Build one chart-data row per machine: `{ name: machine.name, cups: totalCups, revenue: totalRevenue }`. Keep ALL machines in the array (don't filter out zero-activity machines) so the x-axis always shows every machine, matching the reference image.
3. **Date range filter**: add a simple range selector in the card header (reuse `components/ui/popover.tsx` + `components/ui/calendar.tsx`, which already exist in `components/ui/`, to build a from/to date-range picker — there's no existing date-range component to reuse directly, so build a small local one). Default range: last 6 months (or "all time" if simpler to start). Filter transactions by `createdAt` against the selected `[from, to]` before aggregating.
4. **Chart**: Recharts `BarChart` with `<Bar dataKey="cups" yAxisId="left" fill="#22c55e" />` and `<Bar dataKey="revenue" yAxisId="right" fill="#f59e0b" />`, `XAxis dataKey="name"` with angled tick labels (machine names can be long), two `YAxis` (left unlabeled count, right prefixed `₹`), `CartesianGrid`, `Tooltip` with a custom tooltip showing both values per machine, and a legend matching the green/orange colors shown in the reference image (note: this differs from `SalesChart`'s blue/emerald colors — match the reference image's green/orange instead).
5. **Wire it in**: in `app/(private)/dashboard/page.tsx`, inside the `selectedMachine === "all"` branch, render `<MachineComparisonChart machines={machines} />` in its own full-width `<div>`, placed after the existing `<SalesChart />` block (around line 139) and before the status overview cards grid (around line 142). Don't touch `SalesChart` itself — it serves a different purpose (aggregate trend over time vs. this new per-machine comparison).

## Constraints

- Don't modify `convex/schema.ts` or add new Convex functions — `machines.list` and `transactions.list` already provide everything needed client-side.
- Don't touch anything under `app/(private)/admin`, `components/kitchen-sections/`, or the per-machine detail views (`components/machine-details/*`) — this is strictly an addition to the all-machines overview dashboard.
- Match existing code style: TypeScript, `"use client"`, Tailwind classes, `framer-motion` entrance animation consistent with sibling cards in `DashboardPage`, dark-mode classes (`dark:bg-...`) throughout.
- No backend changes, no new dependencies — `recharts`, `date-fns`, and the shadcn `Popover`/`Calendar` primitives are already installed/used elsewhere in the repo.
